Tutorial de GDB

Joar Esteban Buitrago Carrillo

¿Qué es GDB?

GDB es el acrónimo de GNU DeBugger.

Es el depurador estándar de GCC para los lenguajes de programación de C, C++ y Fortran escrito por Richard Stallman en 1986.

Un bug es un error que no se entiende de donde surge, nace de encontrar un insecto en una maquina en 1947 por Grace Murray Hopper (Física y Matemática). GDB funciona para Ada, C, C++, Objective-C, Free Pascal, Fortran, Go y otros muy parcialmente.

Preparación

Antes de trabajar con GDB es necesario compilar el código de manera tal que sea entendible para GDB. 

          g++ -g codigo.cpp
La bandera de compilación con información de debug es -g, esta mantiene la ubicación de los símbolos y el código, esto hace que el ejecutable también sea más grande.

Preparación

En caso de el uso de CMake, es necesario definir la variable CMAKE_BUILD_TYPE con el valor de Debug. 

          set(CMAKE_BUILD_TYPE Debug)
          cmake -DCMAKE_BUILD_TYPE=Debug ..
Los códigos son de ejemplo, y pueden variar.

Invocación

Para iniciar GDB es necesario solamente llamarlo desde la línea de comandos, utilizando como argumento el nombre del ejecutable. 

          gdb nombre_del_ejecutable

En caso de que el ejecutable reciba argumentos desde la línea de comandos, se debe llamar con la opción --args seguido de todos los argumentos de entrada. 

          gdb --args nombre_del_ejecutable argumento_1 ... argumento_n
GDB solamente funciona desde la línea de comandos, no tiene una interfaz grafica propia.

Invocación

Tambien es posible hacer que GDB lea un programa utilizando el comando file. 

          file nombre_del_ejecutable

En este caso, para llamar los argumentos del programa es necesario utilizar el comando set args 

          set args argumento_1 ... argumento_n

Comandos

Por la forma en como funciona GDB es necesario el uso de comandos que entran por medio de su interfaz.

  • next: Avanza a la siguiente instrucción en el «scope» actual.
  • step: Avanza un paso en la ejecución.
  • backtrace: Muestra los registros de lo que se ha ejecutado desde el inicio del programa.
  • print: Imprime sus argumentos a pantalla.
  • set: Asigna valores a variables.
  • etc.
